11 - Clases

¿Por qué usar las clases?

Python es un lenguaje de programación orientado a objetos, lo que significa que manipula construcciones de programación llamadas objetos. Se puede pensar en un objeto como una estructura de datos única que contiene tanto datos como funciones; las funciones de un objeto se llaman sus métodos. Por ejemplo, cada vez que se llama

len("Eric")

Python está comprobando si el objeto de cadena que le pasaste tiene una longitud, y si la tiene, devuelve el valor asociado a ese atributo. Cuando llamas a

mi_dicto.items()

Python comprueba si mi_dicto tiene un método items() (que tienen todos los diccionarios) y ejecuta ese método si lo encuentra.

¿Pero qué hace que "Eric" sea una cadena y mi_dicto un diccionario? El hecho de que sean instancias de las clases str y dict, respectivamente. Una clase es sólo una forma de organizar y producir objetos con atributos y métodos similares.

Sintaxis de clase

Una clase básica consiste sólo en la palabra clave de la clase, el nombre de la clase y la clase de la que la nueva clase hereda entre paréntesis. (Pronto llegaremos a la herencia.) Por ahora, nuestras clases heredarán de la clase objeto, así:

clase NewClass( object):
  # La magia de la clase aquí

Esto les da los poderes y habilidades de un objeto Python. Por convención, los nombres de clase Python definidos por el usuario comienzan con una letra mayúscula.

Habrán notado en nuestro ejemplo del primer ejercicio que comenzamos nuestra definición de clase con una función de aspecto extraño: init(). Esta función es necesaria para las clases, y se usa para inicializar los objetos que crea. init() siempre toma al menos un argumento, self, que se refiere al objeto que se está creando. Se puede pensar en init() como la función que "arranca" cada objeto que la clase crea.

Instanciar (crear) nuestro primer objeto.

Hasta ahora, init() sólo toma un parámetro: self. Esta es una convención de Python; no hay nada mágico en la palabra self. Sin embargo, es abrumadoramente común usar self como el primer parámetro en init(), así que deberías hacer esto para que otras personas entiendan tu código.

La parte que es mágica es el hecho de que self es el primer parámetro pasado a init(). Python usará el primer parámetro que init() recibe para referirse al objeto que se está creando; por eso a menudo se llama self, ya que este parámetro le da al objeto que se está creando su identidad.

class Square(object):
  def init(self):
    self.sides = 4

my_shape = Square()
print (my_shape.sides)
class Square(object):
  def init(self):
    self.sides = 4

my_shape = Square()
print (my_shape.sides)
  1. Primero creamos una clase llamada Square con un atributo de lados.
  2. Fuera de la definición de la clase, creamos una nueva instancia de Square llamada my_shape y accedemos a ese atributo usando my_shape.sides.

Más sobre init() y self

Ahora que empiezas a entender cómo funcionan las clases y los objetos, vale la pena profundizar un poco más en el init() y el self. ¡Pueden ser confusos!

Como se ha mencionado, puedes pensar en init() como el método que "arranca" un objeto de instancia de clase: el bit init es la abreviatura de "initialize".

El primer argumento init() gets se utiliza para referirse al objeto de la instancia, y por convención, ese argumento se llama self. Si añades argumentos adicionales -por ejemplo, un nombre y una edad para tu animal- estableciendo cada uno de ellos igual a self.name y self.age en el cuerpo de init() hará que cuando crees un objeto de instancia de tu clase Animal, necesites darle a cada instancia un nombre y una edad, y esos estarán asociados con la instancia particular que crees.

Ámbito de la clase

Otro aspecto importante de las clases de Python es el alcance. El alcance de una variable es el contexto en el que es visible para el programa.

Puede sorprenderte saber que no todas las variables son accesibles a todas las partes de un programa Python en todo momento. Cuando se trata de clases, puede haber variables que están disponibles en todas partes (variables globales), variables que sólo están disponibles para los miembros de una determinada clase (variables de miembros) y variables que sólo están disponibles para instancias particulares de una clase (variables de instancia).

Lo mismo ocurre con las funciones: algunas están disponibles en todas partes, otras sólo están disponibles para los miembros de una cierta clase, y otras sólo están disponibles para objetos de instancia particular.

Una clase puede tener cualquier número de variables de miembros. Estas son variables que están disponibles para todos los miembros de una clase.

hippo = Animal("Jake", 12)
cat = Animal("Boots", 3)
print (hippo.is_alive)
hippo.is_alive = False
print (hippo.is_alive)
print (cat.is_alive)
hippo = Animal("Jake", 12)
cat = Animal("Boots", 3)
print (hippo.is_alive)
hippo.is_alive = False
print (hippo.is_alive)
print (cat.is_alive)
  1. En el ejemplo anterior, creamos dos instancias de Animal.
  2. Luego imprimimos True, el valor por defecto almacenado en la variable de miembro de hipopótamo is_alive.
  3. A continuación, lo ponemos en False y lo imprimimos para asegurarnos.
  4. Finalmente, imprimimos True, el valor almacenado en la variable de miembro is_alive del gato. Sólo cambiamos la variable en hipopótamo, no en gato.

La herencia es un concepto difícil, así que vamos a ir paso a paso.

La herencia es el proceso por el cual una clase toma los atributos y métodos de otra, y se usa para expresar una relación is-a. Por ejemplo, un Panda es un oso, así que una clase de Panda podría heredar de una clase de Oso. Sin embargo, un Toyota no es un Tractor, por lo que no debería heredar de la clase Tractor (aunque tengan muchos atributos y métodos en común). En su lugar, tanto Toyota como Tractor podrían heredar en última instancia de la misma clase de Vehículo.

class Customer(object):
  """Produces objects that represent customers."""
  def init(self, customer_id):
    self.customer_id = customer_id

  def display_cart(self):
    print ("I'm a string that stands in for the contents of your shopping cart!")

class ReturningCustomer(Customer):
  """For customers of the repeat variety."""
  def display_order_history(self):
    print ("I'm a string that stands in for your order history!")

monty_python = ReturningCustomer("ID: 12345")
monty_python.display_cart()
monty_python.display_order_history()

Sintaxis de la herencia

En Python, la herencia funciona así:

clase DerivedClass( BaseClass):
	# El código va aquí

donde DerivedClass es la nueva clase que estás haciendo y BaseClass es la clase de la que esa nueva clase hereda.

¡Anular!

A veces querrás que una clase que herede de otra no sólo adopte los métodos y atributos de su padre, sino que anule uno o más de ellos.

class Employee(object):
  def init(self, name):
    self.name = name
  def greet(self, other):
    print ("Hello, %s" % other.name)

class CEO(Employee):
  def greet(self, other):
    print ("Get back to work, %s!" % other.name)

ceo = CEO("Emily")
emp = Employee("Steve")
emp.greet(ceo)
# Hello, Emily
ceo.greet(emp)
# Get back to work, Steve!

Por otro lado, a veces trabajarás con una clase (o subclase) derivada y te darás cuenta de que has sobrescrito un método o atributo definido en la clase base de esa clase (también llamada parentela o superclase) que realmente necesitas. No tengas miedo! Puedes acceder directamente a los atributos o métodos de una superclase con la superllamada incorporada de Python.

La sintaxis tiene este aspecto:

class Derived(Base):
  def m(self):
    return super(Derived, self).m()
class Derived(Base):
  def m(self):
    return super(Derived, self).m()

Donde m() es un método de la clase base.

La clase base

Las clases pueden ser muy útiles para almacenar objetos complicados con sus propios métodos y variables. Definir una clase es muy parecido a definir una función, pero usamos la palabra clave de la clase en su lugar. También usamos la palabra objeto entre paréntesis porque queremos que nuestras clases hereden la clase objeto. Esto significa que nuestra clase tiene todas las propiedades de un objeto, que es la clase más simple y básica. Más tarde veremos que las clases pueden heredar otras clases más complicadas. Una clase vacía se vería así:

clase ClassName(object):
	# Las declaraciones de clase van aquí

Crear una instancia de una clase

Podemos usar clases para crear nuevos objetos, que decimos son instancias de esas clases.
Crear una nueva instancia de una clase es tan fácil como decir:

newObject = ClassName()

Las variables de los miembros de la clase

Las clases pueden tener variables de miembros que almacenan información sobre cada objeto de la clase. Las llamamos variables miembros ya que son información que pertenece al objeto de clase.

Crear variables miembros y asignarles valores iniciales es tan fácil como crear cualquier otra variable:

class ClassName(object):
	memberVariable = "initialValue"

Llamando a las variables de los miembros de la clase

Cada objeto de clase que creamos tiene su propio conjunto de variables miembros. Como hemos creado un objeto my_car que es una instancia de la clase Car, my_car ya debería tener una variable miembro llamada condition. A este atributo se le asigna un valor tan pronto como se crea my_car.

class Car(object):
  condition = "new"
my_car = Car()

print (my_car.condition)

class Car(object):
  condition = "new"
my_car = Car()

print (my_car.condition)

Iniciando una clase

Hay una función especial llamada init() que se llama cada vez que creamos una nueva instancia de una clase. Existe por defecto, aunque no la veamos. Sin embargo, podemos definir nuestra propia función init() dentro de la clase, sobrescribiendo la versión por defecto. Puede que queramos hacer esto para proporcionar más variables de entrada, como lo haríamos con cualquier otra función.

El primer argumento que se pasa a init() debe ser siempre la palabra clave self - así es como el objeto se sigue a sí mismo internamente - pero podemos pasar variables adicionales después de eso.

Para asignar una variable a la clase (creando una variable miembro), utilizamos la notación por puntos. Por ejemplo, si pasamos newVariable a nuestra clase, dentro de la función init() diríamos:

self.new_variable = new_variable

Refiriéndose a las variables de los miembros

Llamar a las variables de los miembros de la clase funciona igual tanto si esos valores se crean dentro de la clase (como el estado de nuestro coche) como si los valores se pasan al nuevo objeto en la inicialización. Utilizamos la notación de puntos para acceder a las variables miembros de las clases, ya que esas variables pertenecen al objeto.

Por ejemplo, si hubiéramos creado una variable miembro llamada new_variable, una nueva instancia de la clase llamada new_object podría acceder a esta variable diciendo:

new_object.new_variable

Creando métodos de clase

Además de las variables de los miembros, las clases también pueden tener sus propios métodos. Por ejemplo:

class Square(object):
  def init(self, side):
    self.side = side

  def perimeter(self):
    return self.side * 4
class Square(object):
  def init(self, side):
    self.side = side

  def perimeter(self):
    return self.side * 4

El método de la clase perimeter() es idéntico a la definición de cualquier otra función, excepto que está escrito dentro de la definición de la clase Square.

Al igual que cuando definimos init(), es necesario proporcionar self como primer argumento de cualquier método de clase.

Modificación de las variables miembros

Podemos modificar las variables que pertenecen a una clase de la misma manera que inicializamos esas variables miembros. Esto puede ser útil cuando queremos cambiar el valor que toma una variable en base a algo que ocurre dentro de un método de clase.

Herencia

Uno de los beneficios de las clases es que podemos crear clases más complicadas que heredan las variables o métodos de sus clases parentales. Esto nos ahorra tiempo y nos ayuda a construir objetos más complicados, ya que estas clases hijas también pueden incluir variables o métodos adicionales.

Definimos una clase "hija" que hereda todas las variables y funciones de su clase "madre" de esta manera:

clase ChildClass(ParentClass):
	# Las nuevas variables y funciones van aquí

Normalmente utilizamos el objeto como clase madre porque es el tipo más básico de clase, pero al especificar una clase diferente, podemos heredar una funcionalidad más complicada.

Métodos de anulación

Como nuestro ElectricCar es un tipo de coche más especializado, podemos darle al ElectricCar su propio método drive_car() que tiene una funcionalidad diferente a la de la clase de coche original.

class Car(object):
  condition = "new"
  def init(self, model, color, mpg):
    self.model = model
    self.color = color
    self.mpg   = mpg
   
  def display_car(self):
    print "This is a %s %s with %s MPG." % (self.color, self.model, str(self.mpg))
    
  def drive_car(self):
    self.condition = "used"
    
class ElectricCar(Car):
  def init(self, model, color, mpg, battery_type):
    self.model = model
    self.color = color
    self.mpg   = mpg
    self.battery_type = battery_type
    
  def drive_car(self):
    self.condition = "like new"

my_car = ElectricCar("DeLorean", "silver", 88, "molten salt")

print (my_car.condition)
my_car.drive_car()
print (my_car.condition)

Construyendo clases útiles

Lo más probable es que no diseñes clases de auto en el mundo real en un futuro cercano. Normalmente, las clases son más útiles para mantener y acceder a colecciones abstractas de datos.

Un método de clase útil para anular es el método incorporado repr(), que es la abreviatura de representación; al proporcionar un valor de retorno en este método, podemos decirle a Python cómo representar un objeto de nuestra clase (por ejemplo, cuando se utiliza una declaración print).
