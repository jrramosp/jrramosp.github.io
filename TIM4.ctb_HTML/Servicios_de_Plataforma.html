Los servicios de plataforma ofrecen una plataforma para que los desarrolladores codifiquen, compilen y administren aplicaciones de software.

Configurar servicios de comunicación

El primero es Internet Chat Relay o IRC, que es un protocolo usado para mensajes de chat. IRC opera en un modelo cliente/servidor, por lo tanto, hay mucho software cliente IRC que puede usarse para conectarse a un servidor IRC.
También hay otros protocolos de comunicaciones, llamados protocolos abiertos de mensajería instantánea (MI), que se usan ampliamente y están integrados en diferentes aplicaciones de comunicación. Uno de los protocolos de comunicación más populares es XMPP o Protocolo extensible de mensajería y presencia. Es un protocolo de código abierto usado en aplicaciones de mensajería instantánea y servicios de redes sociales. XMPP se usa incluso en aplicaciones de Internet de las cosas, entre otros casos de uso.

Configuración de servicios de correo electrónico

Hay dos formas de configurar el correo electrónico para una compañía. La primera es ejecutar tu propio servidor administrado. Con esta opción, configuras el software del servicio de correo electrónico en un servidor, luego creas un registro DNS para tu servidor de correo.
Hay diferentes registros de DNS. Recuerda que el registro A se usa para nombres de host, pero en servidores de correo electrónico, usamos MX para el registro de intercambio de correo.
Un enfoque alternativo a configurar tu propio servicio de correo electrónico es usar un proveedor de servicios de correo electrónico, como Google Suite.
El Protocolo de oficina de correos, o POP versión 3, es un protocolo de correo electrónico que descarga el correo electrónico de un servidor de correo electrónico a tu dispositivo local. Luego, borra el correo electrónico. del servidor correspondiente. Si quieres recuperar tu correo electrónico a través de POP3, solo puedes verlo desde un dispositivo. Si te preocupan las limitaciones de almacenamiento y la seguridad, tal vez quieras considerar el uso de POP3 sobre algo como IMAP.
Hablando de IMAP, o Protocolo de acceso a mensajes de Internet, te permite descargar correos electrónicos de tu servidor de correo electrónico en múltiples dispositivos. Mantiene tus mensajes en el servidor de correo electrónico.
La última es el Protocolo de transferencia simple de correo, o SMTP, que es un protocolo que se usa para enviar correos electrónicos.

Configurar servicios de seguridad

HTTPS, o Protocolo seguro de transferencia de hipertexto, es una versión segura de HTTP. Asegura que la comunicación de tu navegador web con el sitio web sea segura a través de la encriptación. A HTTPS también se lo conoce como HTTP sobre TLS o HTTP sobre SSL. Esto es porque hay dos protocolos que nos permiten hacer que nuestros servicios web sean seguros.
El primero es el Protocolo de seguridad de la capa de transporte, o TLS, que es la forma más popular de mantener comunicaciones seguras a través de una red. TLS se usa ampliamente para mantener una navegación web segura, pero se puede utilizar en muchas otras aplicaciones también.
El segundo protocolo es el Protocolo de capa de conexión segura, o SSL. Es una forma de asegurar la comunicación entre un servidor web y un cliente, pero es bastante antiguo e inseguro, por lo que otras opciones como TLS lo vienen reemplazando.
Los dos protocolos a menudo se usan indistintamente. De hecho, SSL versión 3.0 era básicamente la versión 1.0 de TLS. Pero las nuevas características y las actualizaciones de TLS lo han hecho más seguro que SSL.
Si estás administrando el sitio web de una organización en un servidor, ¿cómo habilitas TLS en el servidor para que el sitio pueda estar usando HTTPS? Bueno, necesitas conseguir un certificado digital de confianza de una entidad llamada autoridad de certificación. La autoridad de certificación le otorga a tu sitio web un certificado que dice que confía en que controlas el servidor web. Y verifica que eres quien dices ser. Una vez que hace eso, puedes instalar el certificado en tu servidor web. Así, cuando los usuarios visiten tu sitio, verán HTTPS en la URL en lugar de HTTP.

Almacenamiento de archivos de red

El sistema de archivos de red, o NFS, nos permite hacer esto. Es un protocolo que permite compartir archivos a través de una red. Puedes instalar software de servidor NFS que modifique los archivos de configuración de los directorios a los que deseas permitir el acceso compartido. Una vez que hagas eso, el servicio NFS se ejecutará en el servidor, en segundo plano. En cada máquina cliente que quiera acceder a un servidor, simplemente montas el sistema de archivos como lo harías con cualquier otro sistema de archivos. Hay una salvedad: deberás usar el nombre de host en lugar de un dispositivo de disco físico.
Los servicios Samba son similares a los de NFS, ya que pueden compartir y administrar servicios de archivos de forma centralizada. Además, todos los principales sistemas operativos pueden usar el intercambio de archivos Samba. La única razón por la que te podría interesar considerar Samba sobre NFS es porque funciona mejor con los sistemas operativos Windows. También incluye otros servicios que pueden integrarse con tu organización, como los servicios de impresión.
SMB es un protocolo que implementa Samba.
Una solución relativamente asequible para el hardware de almacenamiento FAT es usar un almacenamiento conectado a la red, o NAS. En lugar de configurar un servidor exclusivo como lo harías con otros servicios, NAS son computadoras optimizadas para el almacenamiento de archivos. Por lo general, vienen con un sistema operativo más despejado solo para entregar archivos a través de una red.

¿Qué es un servidor de base de datos?

La información del cliente, como artículos de noticias, videos, grandes cantidades de texto, archivos de imagen o de audio, generalmente se almacena en una base de datos. Las bases de datos nos permiten almacenar consultas, y filtrar y administrar grandes cantidades de datos.
Los servidores de bases de datos consisten en software de base de datos en ejecución en el que puedes leer y escribir. Sistemas de bases de datos comunes como MySQL y PostgreSQL se usan ampliamente en desarrollo web y de aplicaciones y en estadísticas de datos. Estos sistemas de bases de datos, en general, requieren un conocimiento de lenguajes o sintaxis especiales para poder analizar y filtrar a través de grandes cantidades de datos.

¿Está el sitio web abajo?

Los códigos de estado HTTP son códigos o números que indican algún tipo de error o mensajes de información surgidos al intentar acceder a un recurso web.
Un error 404 indica que la URL que ingresaste no apunta a ningún sitio.
Los navegadores actuales han incorporado herramientas que nos ayudan a diagnosticar problemas con un navegador web o un sitio web en sí. La Herramientas para desarrolladores son un gran recurso para probar y depurar problemas con el sitio web o el navegador.
Los códigos de estado HTTP que comienzan con 4xx indican un problema del lado del cliente. El cliente intenta hacer algo que no pudo, como ingresar una URL incorrecta. acceder a algo para lo que no tenía autorización, etc.
Los otros códigos de estado HTTP comunes que puedes ver comenzar con 5xx. Estos errores indican un problema del lado del servidor. El servidor web que aloja este contenido web está experimentando problemas y, con suerte, los administradores lo están investigando.
También nos pueden indicar cuándo nuestra solicitud es correcta, lo que se indica con códigos que comienzan con 2xx.
