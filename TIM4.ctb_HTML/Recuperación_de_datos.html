¿Qué es la recuperación de datos?

La recuperación de datos, en términos generales, es el proceso que intenta restaurar datos después de un evento inesperado que provoca su pérdida o corrupción.
Cómo tratar de restaurar estos datos perdidos depende de algunos factores. Uno es la naturaleza de los datos perdidos. Si un dispositivo se dañó, podrías recuperar los datos desde el hardware dañado. Esto podría implicar el uso de software recuperación de datos, que puede analizar discos duros o unidades flash con fallas y tratar de localizar y extraer archivos de datos. Otro factor que afectaría tu recuperación de datos es la presencia de copias de seguridad. Si tienes suerte, o si tuviste la previsión de planificar eventos inesperados, tienes copias de seguridad de los datos. Y puedes restaurar los datos que se perdieron.
La mejor manera de estar preparado para un evento de pérdida de datos es implementar un plan y procedimiento ante desastres bien concebido. Los planes de desastre deben involucrar la obtención regular de copias de seguridad de todos los datos críticos necesarios para tus procesos comerciales en curso.
Imagina que algo salió mal con tus sistemas y tuviste que usar un plan ante desastres. Es posible que al recuperar tus datos, descubras problemas que no estaban contemplados en el plan ante desastres. Una autopsia es una forma de documentar cualquier problema descubierto en el camino y, lo más importante, la manera en que los solucionaste para asegurarte de que no vuelvan a suceder.

Copia de seguridad de tus datos

Lo primero que debes averiguar es para qué datos necesitas crear una copia de seguridad. En un mundo perfecto, solo debes hacer copias de seguridad de los datos absolutamente necesarios para las operaciones y que puedan hallarse en otra fuente.
Crear una copia de seguridad de los datos no es gratuito. Cada archivo adicional de copia de seguridad ocupa algo más de espacio en el disco, lo que aumenta los costos generales de tu solución de copia de seguridad. Una vez que hayas averiguado a qué datos querrías crearles una copia de seguridad, debes averiguar la cantidad total de datos que tienes actualmente. Pero no basta con pensar en cuáles son tus requisitos de almacenamiento de copia de seguridad en este momento. Tu organización puede seguir creciendo y tus necesidades de copiado de seguridad deberían crecer con ella. Asegúrate de tener en cuenta el crecimiento a futuro y elige una solución lo suficientemente flexible para dar fácil cabida a los aumentos en el copiado de seguridad de datos.
Se puede hacer una copia de seguridad local de los datos a los sistemas en el sitio, o la copia de seguridad puede enviarse externamente a sistemas remotos.
La ventaja de las soluciones de copiado de seguridad en el sitio es que los datos están físicamente muy cerca. Esto hace que el acceso a los datos sea mucho más rápido. No necesitarás tanto ancho de banda saliente ya que no estás enviando los datos fuera de tu red interna. Si necesitas restaurar datos desde copias de seguridad, eso debería ser bastante rápido, ya que los datos están a la mano.
Las copias de seguridad externas nos preparan mejor para eventos catastróficos que pueden borrar datos de una oficina completa, pero enviar datos fuera del sitio significa que debes transmitir los datos fuera de tu red. Esto implica considerar cosas como la encriptación y el ancho de banda.
La copia de seguridad de datos resultante que se almacene también debe encriptarse en reposo.

Soluciones de copia de seguridad

Una cosa que debes tener en cuenta al evaluar la estrategia de copiado de seguridad de una organización es el período de almacenamiento de copias de seguridad. ¿Por cuánto tiempo debes conservar las copias de seguridad? La respuesta a esta pregunta afectará tu necesidad de almacenamiento a largo plazo y los costos generales de mantenimiento de un sistema de copias de seguridad. Un enfoque que equilibra el costo con la conveniencia es archivar los datos más antiguos usando un mecanismo de almacenamiento más lento pero más barato. El medio estándar para almacenar copias de seguridad de datos en archivos es la cinta de datos. Se parece mucho a las cintas de audio, ya que ambas usan carretes de cinta magnética a través de máquinas que permiten leer y escribir datos de la cinta. El almacenamiento en cinta es bastante económico, pero no es tan sencillo ni tan rápido de acceder como los datos almacenados en discos duros o unidades de estado sólido.
Una es la utilidad de línea de comandos rsync. "rsync" no es una herramienta de copiado de seguridad explícita, pero comúnmente se la usa como tal. Es una utilidad de transferencia de archivos diseñada para transferir y sincronizar archivos entre ubicaciones o equipos de manera eficiente. Admite compresión y puedes usar SSH para transferir datos a través de una red en condiciones seguras. Con SSH, también puedes sincronizar archivos entre máquinas remotas, lo cual es muy útil para crear copias de seguridad simples y automatizadas.
https://wiki.archlinux.org/index.php/rsync#As_a_backup_utility
Apple tiene una solución de copiado de seguridad de primera mano disponible para sus SO Mac, llamada Time Machine. Opera mediante un modelo de copia de seguridad incremental. Time Machine admite la restauración de todo un sistema, desde la copia de seguridad o de archivos individuales. Incluso permite restaurar versiones más antiguas de archivos de copia de seguridad.
https://support.apple.com/en-us/HT201250
Microsoft también ofrece una solución de primera mano llamada Copias de seguridad y restauración. Funciona de dos modos: como una versión basada en archivos en la que se crea una copia de seguridad en un archivo zip, o como una imagen del sistema en la que se guarda todo el disco, bloque por bloque, en un archivo. El copiado de seguridad basado en archivos admite copias de seguridad completas o incrementales. El copiado de seguridad mediante la imagen del sistema admite el modo diferencial, en el que solo se crean copias de seguridad de los bloques en disco que se modificaron desde la última copia de seguridad.
https://support.microsoft.com/en-us/help/17127/windows-back-up-restore

Pruebas de copias de seguridad

Hay un último tema importantísimo cuando se trata de copias de seguridad. Someterlas a prueba. La enseñanza aquí es que no basta con configurar copias de seguridad a intervalos regulares, eso es sólo la mitad de la ecuación. La otra mitad es un proceso de recuperación y ese proceso debe probarse con regularidad. Los procedimientos de restauración deben estar documentados y ser accesibles para que cualquier persona con el acceso correcto puede restaurar las operaciones cuando sea necesario.
Entonces, documenta el procedimiento y verifica la documentación para asegurarte de que funcione ahora y en el futuro. Este proceso se llama prueba de recuperación ante desastres y es fundamental para garantizar que un sistema de recuperación funcione bien. Las pruebas de recuperación ante desastres deben ser un ejercicio regular que se realice una vez al año aproximadamente. Deben incluir diferentes equipos, entre ellos, los especialistas de soporte de TI, quienes pasarán por simulaciones de eventos de desastre.
Estos eventos simulados son la manera perfecta de descubrir cualquier brecha en tu planificación. Si descubres que no estás protegido contra la pérdida de datos en una situación determinada, es una oportunidad para aprender y corregir esta brecha sin correr el riesgo de perder datos reales.

Tipos de copia de seguridad

¿Pero cómo decides exactamente cómo y cuándo hacer una copia de seguridad de los datos?
Puedes hacer una copia de seguridad completa con frecuencia, lo que implica copiar los datos que necesitan una copia de seguridad completa. Este mecanismo incluye el contenido completo de todos los archivos cuya copia de seguridad debe crearse, sin importar si los datos se modificaron o no. En el caso de datos que no cambian muy a menudo, como los archivos de configuración del sistema operativo, este enfoque puede ser ineficiente. Estás haciendo muchas copias de seguridad de datos que no cambian, lo que desperdicia espacio y consume ancho de banda innecesariamente.
Un enfoque más eficiente es solo crear copias de seguridad de los archivos que se modificaron o se crearon desde la última copia de seguridad completa. A esto se lo llama copia de seguridad diferencial. La ventaja es que no almacenas copias de seguridad de datos invariables duplicados, solo los archivos que se modificaron. Ahorrarnos cierto espacio de almacenamiento y tiempo para crear la copia de seguridad. Con el tiempo, terminas registrando y almacenando muchas copias de archivos que cambian mucho, lo que también ocupará cada vez más espacio en disco con el tiempo.
Para evitar esto, se recomienda realizar copias de seguridad completas no frecuentes, al tiempo que se crean copias de seguridad diferenciales con más frecuencia. ¿Con qué frecuencia se realiza una copia de seguridad completa? Depende de la amplitud del período de cambios que desees registrar.
Otra forma eficiente de crear copias de seguridad de datos variables es realizar copias de seguridad incrementales regularmente. Mientras que una copia de seguridad diferencial respalda archivos que han sido modificados o creados desde la última copia de seguridad completa, una copia de seguridad incremental es cuando solo los datos que se modifican en los archivos desde la última copia de seguridad incremental se respaldan. Esto es aún más eficiente en términos de espacio en disco y de tiempo requerido en comparación con las copias de seguridad diferenciales.
Una vez más, deberás hacer copias de seguridad incrementales frecuentes y copias de seguridad completas menos frecuentes. Pero como este enfoque solo resuelve diferencias en los archivos que se modificaron desde la última copia de seguridad incremental, es posible que se necesiten todas las copias de seguridad incrementales para reconstruir los archivos por completo. Si una de estas copias de seguridad incrementales falta o sufre corrupción, tal vez no sea posible recuperar datos más recientes que los de la última copia de seguridad completa. Otra desventaja es que la recuperación podría llevar más tiempo. Esto es porque se tiene que recrear la versión más reciente de los datos en la copia de seguridad mediante la integración de la última copia de seguridad completa con cada copia de seguridad incremental que le sigue.
En el caso de archivos gigantes que cambian con frecuencia, esto podría requerir mucho tiempo de procesamiento. Una cosa más que pueden hacer los sistemas de respaldo para ayudar a ahorrar espacio es la compresión de archivos. Al crear una copia de seguridad, todas las estructuras de archivos y carpetas se copiarán y pondrán en un archivo comprimido. Los archivos comprimidos son útiles para mantener los archivos organizados y conservar la estructura completa. Las copias de seguridad, además de contener archivos, pueden comprimirse. Este mecanismo almacena los mismos datos, pero requiere menos espacio en disco mediante el uso de algoritmos complejos. Esto significa que el ahorro de espacio por compresión dependerá del tipo de archivo incluido en la copia de seguridad. Otra cosa que debes conocer sobre la compresión de copias de seguridad es el costo de restauración. Para recuperar datos de una copia de seguridad, primero hay que descomprimirlos. Según el tamaño de tus copias de seguridad, esto podría requerir mucho tiempo y espacio en disco para la expansión.
RAID, que significa matriz redundante de discos independientes, es un método que toma muchos discos físicos y los combina en un gran disco virtual. Hay muchos tipos de configuraciones RAID, llamadas niveles. Según las características que se deseen de la RAID, diversos niveles de RAID priorizan características como el desempeño, la capacidad o la confiabilidad. Las matrices RAID son una forma excelente y económica de crear mucha capacidad de datos al tiempo que se minimiza el riesgo de pérdida de datos en caso de fallo del disco. Incluso pueden ser lo suficientemente flexibles para permitir la ampliación de la capacidad del disco a futuro.
RAID no es una solución de copiado de seguridad, sino una solución de almacenamiento de datos que dispone de cierta redundancia ante fallas de hardware en algunos de los niveles RAID. Pero almacenar datos en una matriz RAID no te protege contra la eliminación accidental de archivos ni contra la corrupción de datos a causa del malware.

Plan de recuperación ante desastres

Un plan de recuperación ante desastres es una colección de procedimientos y planes documentados sobre cómo reaccionar y manejar una situación de emergencia o desastre, desde una perspectiva operativa. Esto incluye cosas que deben hacerse antes, durante y después de un desastre. El objetivo del plan de recuperación ante desastres es minimizar la interrupción del negocio y de las operaciones de TI manteniendo el tiempo de inactividad de los sistemas al mínimo y evitando pérdidas significativas de datos.
A pesar de su nombre, un plan de recuperación ante desastres abarcará medidas preventivas y de detección, además del enfoque de recuperación posterior al desastre. Las medidas preventivas abarcan cualquier procedimiento o sistema implementado que minimice de manera proactiva el impacto de un desastre. Esto incluye cosas como copias de seguridad regulares y sistemas redundantes. Todo lo que se hace antes de un desastre real para poder reducir el tiempo de inactividad general del evento se considera preventivo.
La notificación oportuna de un desastre es crítica, ya que algunos pasos del plan de recuperación ante desastres podrían requerir acción urgente para asegurar que no haya pérdida de datos o daños en el equipo.
Otras cosas que deben supervisarse para prevenir cualquier desastre inesperado incluyen las condiciones ambientales dentro de diversas salas de conexión en red. Los sensores de inundación pueden avisarte que hay agua que ingresa al servidor. Los sensores de temperatura y humedad pueden controlar temperaturas peligrosamente altas o niveles óptimos de humedad en el aire. Esto puede alertarte sobre una falla de enfriamiento en la sala de servidores. Los detectores de humo y las alarmas contra incendios también son de importancia crítica.
Los detalles de la evacuación de un edificio por lo general remiten al equipo de administración del edificio, pero como especialista en soporte de TI, es probable que trabajes en estrecha colaboración con algunos miembros de este equipo en asuntos como suministro de energía, sistemas de calefacción y refrigeración y evacuación de edificios.
Por último, las medidas correctivas o de recuperación son las que se aplican después de ocurrido el desastre. Estas medidas implican medidas como restaurar datos perdidos desde las copias de seguridad o recompilar y reconfigurar sistemas dañados.
Cuando un sistema en un par redundante sufre una falla, se lo conoce como único punto de falla. Esto se debe a que ahora, solo se necesita una falla para desactivar completamente el sistema.

Diseño de un plan de recuperación ante desastres

Una buena manera de entender lo que se debe planificar es realizar una evaluación de riesgos. Esto implica estudiar detenidamente las operaciones y las características de tus equipos. Una evaluación de riesgos te permite priorizar ciertos aspectos de las organizaciones que están en mayor riesgo en caso de un evento imprevisto.
Cuando estudies las medidas preventivas, presta atención a los sistemas que carecen de redundancia.
Asegúrate de tener una copia de seguridad y un sistema de recuperación sólidos, junto con una buena estrategia implementada. Idealmente, deberías tener copias de seguridad regulares pero automatizadas en sistemas ubicados en el sitio y fuera de él.
Todo lo que sea crítico para las operaciones debe hacerse redundante siempre que sea posible.
Otra medida preventiva importantísima que se debe evaluar y verificar es la documentación operacional. Asegúrate de que todos los procedimientos operativos importantes estén documentados y sean accesibles. Cualquier medida o detalles de configuración específicos que sean necesarios para restaurar la funcionalidad total de los sistemas y servicios centrales debe documentarse en detalle. Una forma eficaz de hacer esto es verificar periódicamente que los pasos documentados realmente funcionen.
La manera de pensar sobre el diseño de medidas de detección es evaluar qué es lo más crítico para el funcionamiento diario de la organización.
Y, por supuesto, sin lugar a dudas debes someter estos sistemas a pruebas. Simula las condiciones que tus sistemas de supervisión deben detectar por diseño. Asegúrate de que los umbrales de detección disparen las alertas como se supone que deben.
Tu plan de desastre debe incluir referencias o vínculos a documentación para este tipo de tareas.
