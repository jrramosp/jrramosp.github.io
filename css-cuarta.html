Las técnicas de diseño de página CSS nos permiten tomar los elementos contenidos en una página web y controlar dónde están posicionados en relación con su posición predeterminada en el flujo de diseño normal, los otros elementos a su alrededor, su contenedor principal o la ventana / ventana principal.

El flujo normal es la forma en que el navegador establece las páginas HTML de forma predeterminada cuando no hace nada para controlar el diseño de la página.

Los métodos que pueden cambiar cómo se organizan los elementos en CSS son los siguientes:

  • La propiedad display — Valores estandar como block, inline o inline-block puede cambiar cómo se comportan los elementos en flujo normal (ver Types of CSS boxes para mas informacion). Luego tenemos métodos de diseño completos que se activan a través de un valor de display, por ejemplo, grid y flex.
  • Floats — Aplicando un valor float como left puede hacer que los elementos de nivel de bloque se envuelvan a un lado de un elemento, como la forma en que las imágenes a veces tienen texto flotando a su alrededor en diseños de revistas.
  • La propiedad position — Le permite controlar con precisión la ubicación de las cajas dentro de otras cajas. El posicionamiento static es por defecto el de un flujo normal, pero puede hacer que los elementos se distribuyan de forma diferente utilizando otros valores, por ejemplo, siempre fijados en la parte superior izquierda de la ventana del navegador.
  • Table layout — las características diseñadas para diseñar las partes de una tabla HTML se pueden usar en elementos que no son de tabla usando display: table y propiedades asociadas.
  • Multi-column layout — La propiedad Multi-column layout puede hacer que el contenido de un bloque se distribuya en columnas, como puede ver en un periódico.

Flexbox es el nombre corto del Módulo de Disposición de Caja Flexible, diseñado para facilitarnos la disposición de las cosas en una dimensión - ya sea como una fila o como una columna. Para utilizar flexbox, se aplica display: flex al elemento padre de los elementos que se desea diseñar; todos sus hijos directos se convierten en elementos flexibles.

Como ejemplo simple de esto, podemos añadir la propiedad flex a todos los objetos de nuestro hijo, con un valor de 1. Esto hará que todos los objetos crezcan y llenen el contenedor, en lugar de dejar espacio al final.

Grid Layout está diseñado para dos dimensiones - alineando las cosas en filas y columnas. Puede activar Grid Layout con un valor específico de display: grid. Además de usar display: grid, también estamos definiendo algunas líneas y columnas en el padre usando las propiedades grid-template-rows y grid-template-columns respectivamente. No necesito poner ninguna regla sobre los elementos hijo; se colocan automáticamente en las celdas que nuestra red ha creado.

Se puede establecer el inicio y el final de la línea de cada elemento utilizando las propiedades de grid-column y grid-row. Esto hace que los elementos abarquen varias filas o columnas.

La propiedad float de un elemento cambia el comportamiento de ese elemento y de los elementos de nivel de bloque que lo siguen en flujo normal. El elemento se mueve a la izquierda o a la derecha y se retira del flujo normal, y el contenido circundante flota alrededor de la posición flotante. La propiedad float tiene cuatro valores posibles:left, right, none e inherit

El posicionamiento le permite mover un elemento desde donde se colocaría cuando está en flujo normal a otra ubicación. Hay cinco tipos de posicionamiento que debe conocer:

  • El posicionamiento estático es el valor por defecto que recibe cada elemento - sólo significa "poner el elemento en su posición normal en el flujo de diseño del documento - nada especial que ver aquí".
  • El posicionamiento relativo le permite modificar la posición de un elemento en la página, moviéndolo en relación con su posición en flujo normal, incluso superponiéndolo a otros elementos de la página.
  • El posicionamiento absoluto mueve un elemento completamente fuera del flujo normal del diseño de la página, como si estuviera ubicado en su propia capa separada. Desde allí, puede fijarlo en una posición relativa a los bordes del elemento <html> de la página (o su elemento ancestral más cercano). Esto es útil para crear efectos de diseño complejos, como cuadros con pestañas en los que los distintos paneles de contenido se sientan uno encima del otro y se muestran y ocultan como se desee, o paneles de información que se colocan fuera de la pantalla de forma predeterminada, pero que se pueden hacer deslizar en la pantalla mediante un botón de control.
  • El posicionamiento fijo es muy similar al posicionamiento absoluto, excepto que fija un elemento relativo a la ventana del navegador, no otro elemento. Esto es útil para crear efectos como un menú de navegación persistente que siempre permanece en el mismo lugar en la pantalla que el resto de los pergaminos de contenido.
  • El posicionamiento sticky es un nuevo método de posicionamiento que hace que un elemento actúe como position: static hasta que alcanza una posición definida de la ventana de visualización, en cuyo punto actúa como position: fixed.

El posicionamiento relativo, position: relative, le permite desplazar un elemento de la posición en flujo normal que tendría por defecto. Esto significa que puede realizar una tarea como mover un icono un poco hacia abajo para que se alinee con una etiqueta de texto.

El posicionamiento absoluto, position: absolute, se utiliza para eliminar completamente un elemento del flujo normal y colocarlo utilizando desplazamientos desde los bordes de un bloque que lo contiene.

El posicionamiento fijo, position: fixed, elimina nuestro elemento del flujo de documentos de la misma manera que el posicionamiento absoluto. Sin embargo, en lugar de las compensaciones que se aplican desde el contenedor, se aplican desde el Viewport.

El posicionamiento sticky, position: sticky, mezcla el posicionamiento estático por defecto con el posicionamiento fijo. Cuando un objeto tiene position: sticky se desplazará en flujo normal hasta que se desplaza desde la ventana de visualización que hemos definido. En ese momento se "atasca" como si tuviera una position: fixed aplicada.

Para convertir un bloque en un contenedor multicolumna utilizamos la propiedad column-count, que le dice al navegador cuántas columnas nos gustaría tener, o la propiedad column-width, que le dice al navegador que llene el contenedor con tantas columnas de al menos ese ancho.

En Flexbox los siguientes requerimientos para un diseño, son difíciles o imposible de conseguir con las herramientas mencionadas anteriormente de una manera flexible

  • Centrar verticalmente un bloque dentro de su padre
  • Hacer que todos los elementos de un contenedor tengan la misma altura/anchura, independientemente de cuanta altura y anchura esta disponible
  • Hacer que todas las columnas tengan el mismo tamaño, incluso cuando tiene diferente cantidad de contenido.

Para empezar, tenemos que seleccionar qué elementos van a ser colocados como cajas flexibles. Para hacer esto, establecemos un valor especial de display en el elemento padre de los elementos que desea afectar. En este caso queremos presentar los elementos <article>, así que lo establecemos en <section> (que se convierte en un contenedor flexible):


  section {
    display: flex;
  }

Cuando los elementos se colocan como cajas flexibles, se colocan a lo largo de dos ejes:

  • El eje principal o main axis es el eje que corre en la dirección en la que se están colocando los elementos flexibles (por ejemplo, como filas a lo largo de la página, o columnas hacia abajo en la página.) El inicio y el final de este eje se denominan main start y main end (inicio principal y final principal).
  • El eje transversal o cross axis es el eje que corre perpendicular a la dirección en la que se están colocando las piezas flexibles. El inicio y el final de este eje se denominan cross start y cross end.
  • El elemento padre con el display: flex seleccionado (el elemento <section> en nuestro ejemplo) se llama flex container.
  • Los items que se presentan como cajas flexibles dentro del flex container se denominan flex items (los elementos <article> en nuestro ejemplo).

Flexbox proporciona una propiedad llamada flex-direction que especifica en qué dirección corre el eje principal (en qué dirección están dispuestos los hijos de flexbox) - por defecto está configurada como row, lo que hace que estén dispuestos en una fila en la dirección en la que trabaja el idioma predeterminado de su navegador. Puede ser también en column.

Un problema que surge cuando tiene una cantidad fija de ancho o alto en su diseño es que eventualmente sus hijos de la caja flexible desbordarán su contenedor, rompiendo el diseño. Una forma de solucionar esto es añadir la siguiente declaración a la regla de la sección:

flex-wrap: wrap

Ahora tenemos varias filas: se colocan tantos hijos flexbox en cada fila como sea necesario, y cualquier desbordamiento se mueve a la siguiente línea.

En este punto vale la pena señalar que existe una abreviatura de flex-direction y flex-wrap - flex-flow.

flex-flow: row wrap;

Cómo podemos controlar qué proporción de espacio ocupan los ítems de flex.

flex: 1;

Este es un valor de proporción sin unidades que dicta cuánto espacio disponible a lo largo del eje principal ocupará cada elemento flexible. En este caso, le damos a cada <article> elemento un valor de 1, lo que significa que todos ocuparán una cantidad igual del espacio libre que queda después de que se hayan establecido cosas como el padding y margin.

También puede especificar un valor de tamaño mínimo dentro del valor de flex.

flex: 1 200px;

flex es una propiedad abreviada que puede especificar hasta tres valores diferentes:

  • El valor de la proporción sin unidades que discutimos anteriormente. Esto se puede especificar individualmente utilizando la propiedad larga flex-grow.
  • Un segundo valor de proporción sin unidades, el flex-shrink, que entra en juego cuando las piezas flexibles se desbordan de su contenedor. Esto especifica qué cantidad de la cantidad desbordante se retira del tamaño de cada artículo flex, para evitar que se desborde su contenedor.
  • El valor del tamaño mínimo que hemos discutido anteriormente. Esto se puede especificar individualmente utilizando el valor largo de la flex-basis.

También puede utilizar las funciones de flexbox para alinear las posiciones flexibles a lo largo de los ejes principales o cruzados.


   div {
	  display: flex;
	  align-items: center;
	  justify-content: space-around;
	}

align-items controla donde los objetos flex se distribuyen a lo largo del eje transversal.

  • De forma predeterminada, el valor es stretch, que estira todas las posiciones flexibles para rellenar el nivel superior en la dirección del eje transversal. Si el padre no tiene un ancho fijo en la dirección del eje transversal, entonces todas las posiciones flexibles serán tan largas como las posiciones flexibles más largas.
  • El valor center hace que los ítems mantengan sus dimensiones intrínsecas, pero que estén centrados a lo largo del eje transversal.
  • También puede tener valores como flex-start y flex-end, que alinearán todas las posiciones al principio y al final del eje transversal respectivamente.

justify-content controla donde los elementos flexibles se encuentran en el eje principal.

  • El valor por defecto es flex-start, que hace que todas las posiciones se sitúen al principio del eje principal.
  • Puedes usar el flex-end para hacer que se coloquen al final.
  • center es también un valor para justificar el contenido, y hará que los elementos flexibles se sitúen en el centro del eje principal.
  • El valor space-around, es útil - distribuye todos los elementos uniformemente a lo largo del eje principal, con un poco de espacio en cada extremo.
  • Hay otro valor, space-between, que es muy similar a space-around excepto que no deja espacio en ninguno de los extremos.

Flexbox también tiene una función para cambiar el orden de disposición de las posiciones flexibles, sin afectar al orden de origen.

order: 1;
  • Por defecto, todas las posiciones flexibles tienen un valor de orden de 0.
  • Las posiciones de Flex con valores de orden superiores configurados aparecerán más tarde en el orden de visualización que las posiciones con valores de orden inferiores.
  • Los elementos flexibles con el mismo valor del orden aparecerán en su orden de origen. Por lo tanto, si tiene cuatro artículos con valores de orden de 2, 1, 1 y 0 establecidos respectivamente, su orden de visualización sería 4º, 2º, 3º, y luego 1er.
  • La tercera posición aparece después de la segunda porque tiene el mismo valor de orden y está detrás de ella en el orden de origen.
  • Se pueden fijar valores de orden negativos para que las posiciones aparezcan antes que las posiciones con valor de orden 0.

Es posible crear algunos esquemas bastante complejos con flexbox. Está perfectamente bien configurar un artículo flexible para que también sea un contenedor flexible, de modo que sus hijos también estén dispuestos como cajas flexibles.

align-content Esta propiedad alinea las líneas de un contenedor flexible dentro cuando hay espacio extra en el eje transversal, de forma similar a como el justify-content alinea los elementos individuales dentro del eje principal. Es una propiedad que tiene efecto cuando hay más de una línea de artículos flexibles.

align-self permite que la alineación predeterminada (o la especificada por los elementos de alineación) se sustituya para los elementos flexibles individuales.

CSS Grid Layout es un sistema de maquetación bidimensional para la web. Permite distribuir el contenido en filas y columnas, y tiene muchas características que hacen que la construcción de diseños complejos sea sencilla. Una cuadrícula típicamente tendrá columns, rows, y luego espacios entre cada fila y columna - comúnmente referidos como gutters.

Para definir una cuadrícula utilizamos el valor grid de la propiedad display. Al igual que en el caso de Flexbox, esta opción activa el diseño de la cuadrícula y todos los hijos directos del contenedor se convierten en elementos de la cuadrícula.

Para determinar las columnas en una cuadrícula utlizamos la propiedad grid-template-columns. Puede utilizar cualquier unidad de longitud o porcentaje para crear estas líneas de columna. podemos utilizar la unidad fr para dimensionar de forma flexible las filas y columnas de la rejilla. Esta unidad representa una fracción del espacio disponible en el contenedor de la rejilla.

Para crear huecos entre las pistas, usamos las propiedades grid-column-gap para los huecos entre columnas, grid-row-gap para los huecos entre filas, y grid-gap para establecer ambos a la vez. Estos espacios pueden ser cualquier unidad de longitud o un porcentaje, pero no una unidad fr.


     .container {
        display: grid;
        grid-template-columns: 1fr 1fr 1fr;
        grid-gap: 20px;
    }

Puede repetir todo o una parte de su listado de pistas utilizando la notación repeat.

grid-template-columns: repeat(3, 1fr);

Si desea dar a las pistas que el navegador crea un tamaño, puede utilizar las propiedades grid-auto-rows y grid-auto-columns.

La función minmax nos permite establecer un tamaño mínimo y máximo para una pista, por ejemplo minmax(100px, auto). El tamaño mínimo es de 100 píxeles, pero el máximo es automático, que se ampliará para ajustarse al contenido.

grid-auto-rows: minmax(100px, auto);

A veces es útil poder pedir a la cuadrícula que cree tantas columnas como quepan en el contenedor. Hacemos esto estableciendo el valor grid-template-columns usando la notación repeat(), pero en lugar de pasar un número, pasamos la palabra clave auto-fill. Para el segundo parámetro de la función usamos minmax(), con un valor mínimo igual al tamaño mínimo de pista que nos gustaría tener, y un valor máximo.

grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));

Podemos colocar las cosas de acuerdo a estas líneas especificando el inicio y el final de la línea. Hacemos esto usando las siguientes propiedades: grid-column-start, grid-column-end, grid-row-start y grid-row-end. Todas estas propiedades pueden tener un número de línea como valor. También puede utilizar las propiedades abreviadas: grid-column y grid-row. Estos le permiten especificar las líneas de inicio y final a la vez, separadas por un carácter / .

Una forma alternativa de colocar elementos en su cuadrícula es utilizar la propiedad grid-template-areas y darle un nombre a los distintos elementos de su diseño. Luego establecemos los nombres de los elementos con la propiedad grid-area.

Las reglas para grid-template-areas son las siguientes:

  • Necesitas tener todas las celdas de la cuadrícula llenas.
  • Para abarcar dos celdas, repita el nombre.
  • Para dejar una celda vacía, utilice un... (punto).
  • Las áreas deben ser rectangulares - no se puede tener un área en forma de L, por ejemplo.
  • Las áreas no se pueden repetir en diferentes lugares.

Exploremos cómo usar float. El elemento con el float fijado se saca del flujo normal de diseño del documento y se pega a la izquierda o derecha de su contenedor principal. Cualquier contenido que se encuentre por debajo del elemento flotado en el flujo de diseño normal lo rodeará, llenando el espacio del lado derecho o izquierdo del mismo hasta la parte superior del elemento flotado.

Mientras que podemos añadir un margen al float para alejar el texto, no podemos añadir un margen al texto para alejarlo del float. Si queremos evitar que un elemento siguiente se mueva hacia arriba, tenemos que aclararlo. Esto se logra con la característica property.La propiedad clear acepta los siguientes valores:left, right y both.

El posicionamiento le permite sacar elementos del flujo normal de diseño de documentos y hacer que se comporten de forma diferente. El posicionamiento static es el valor por defecto que recibe cada elemento.

El posicionamiento relative es el primer tipo de posición que analizaremos. Esto es muy similar al posicionamiento static, excepto que una vez que el elemento posicionado ha ocupado su lugar en el flujo normal del diseño, puede modificar su posición final, incluso superponiéndolo a otros elementos de la página. top, bottom, left y right se utilizan junto a la posición para especificar exactamente a dónde mover el elemento posicionado.

Un elemento posicionado absolut ya no existe en el flujo normal de disposición de documentos. En su lugar, se asienta sobre su propia capa, separada de todo lo demás. top, bottom, left y right se comportan de manera diferente con un posicionamiento absolute. En lugar de especificar la dirección en la que debe moverse el elemento, especifican la distancia que debe haber entre el elemento y los lados de cada uno de los elementos que lo contienen.

¿Qué elemento es el "elemento de contenido" de un elemento absolutamente posicionado? En resumen, el elemento absolutamente posicionado estará contenido fuera del elemento <html>, y se posicionará en relación con la ventana de visualización inicial.Podemos cambiar el contexto de posicionamiento, es decir, el elemento con el que se posiciona el elemento absolutamente posicionado. Esto se hace colocando la posición relative en uno de los ancestros del elemento - a uno de los elementos dentro de los cuales está anidado.

¿qué determina qué elementos aparecen encima de qué otros elementos? Usando la propiedad z-index.Los valores positivos los elevan hacia arriba en la pila, y los valores negativos los bajan hacia abajo en la pila.

La posición fixed fija un elemento en su lugar en relación con la propia ventana del navegador.

Hay otro valor de posición disponible llamado posición: sticky, que es algo más nuevo que los otros. Se trata básicamente de un híbrido entre la posición relative y la posición fixed, que permite que un elemento posicionado actúe como si estuviera relativamente posicionado hasta que se desplaza a un cierto punto de umbral, después del cual se convierte en fijo.

La indicación de disposición de varias columnas le ofrece un método de disposición del contenido en columnas. Activamos el modo multicol mediante una de las dos propiedades de column-count o column-width. La propiedad column-count creará tantas columnas como el valor que le dé. La propiedad column-width le dará tantas columnas como pueda del tamaño que usted especifique

Se puede dar estilo de la siguiente forma. Cambiar el tamaño del hueco entre columnas utilizando column-gap. Añadir una regla entre columnas con la propiedad column-rule del mismo modo que con la propiedad border.
