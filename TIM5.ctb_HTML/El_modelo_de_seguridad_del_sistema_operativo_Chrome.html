El modelo de seguridad del sistema operativo Chrome

Los principios de seguridad del sistema operativo Chrome


Examinemos el modelo de seguridad de Chrome OS como ejemplo de un sistema que aplica los principios de defensa en profundidad que estuvimos estudiando. Este SO tiene muchas capas diferentes de seguridad diseñadas para protegerse de los ataques. De modo que si una capa se ve en riesgo, las demás siguen vigentes.
Como dijimos en el primer curso, Chrome OS ejecuta Linux bajo el capó y cuenta con una interfaz de usuario simple. La mayor parte de la interacción del sistema ocurre a través del navegador Chrome. El objetivo del sistema operativo es ser una forma segura y simple en la que el usuario interactuará con la Web. El usuario no tiene privilegios administrativos ni permiso para modificar el sistema, ya que el navegador Chrome se ejecuta con permisos de usuario gestionados por el administrador. Esto significa que aun cuando el navegador sea explotado, no se puede manipular el sistema.
Chrome OS, de manera predeterminada, protege el sistema operativo. Esto significa que la configuración del sistema ya está preparada para seguridad y que el usuario no necesita tomar ninguna medida adicional. Chrome OS activa las opciones de fortalecimiento disponibles para todos los sistemas operativos Linux. Pero no todas las distribuciones de Linux habilitan estas opciones.
Repasemos algunas otras características de seguridad específicas desarrolladas para Chrome OS. Como dijimos antes, aplicar actualizaciones de seguridad es fundamental para mantener una infraestructura segura. Chrome OS aborda este problema implementando actualizaciones automáticas. Tan pronto como una actualización esté disponible, el SO la descarga y la instala en segundo plano, sin ninguna interacción del usuario. A los usuarios se les pedirá que reinicien cuando las actualizaciones están disponibles. La actualización comienza a funcionar tan pronto como la máquina se reinicia.
Una de las características de seguridad de Chrome OS se llama zona de pruebas (sandboxing). Cada pestaña en un navegador Chrome, así como cada uno de los servicios del sistema, se ejecuta en un proceso separado, completamente independiente de las demás. Por lo tanto, si el usuario visita un sitio web malicioso, la mayor parte del daño que el sitio web puede hacer se limita a esa sola pestaña. Sólo los datos de sesión almacenados en el disco duro de la computadora, como las cookies, el caché y los marcadores son accesibles a todas las pestañas. Y, por supuesto, los datos de sesión creados por un sitio web no están a disposición de otros sitios web.
Otra característica de seguridad importante de Chrome OS es el modo de recuperación automático y fácil de usar. Cuando el sistema detecta un problema, ya sea debido a la corrupción de datos o a un atacante que manipula el sistema operativo instalado, este pasará automáticamente al modo de recuperación y ayudará al usuario a devolver el sistema a un estado de funcionamiento confiable.
Powerwash es otra característica de seguridad de Chrome OS. Permite al usuario restablecer rápidamente la máquina a sus valores predeterminados de fábrica. Como todos los datos personales de los usuarios se almacenan en la nube, esta característica eliminará todo archivo descargado, cookie y caché almacenado en la unidad de disco duro de la computadora. Poder borrar todo desde unidades locales mejora la seguridad al viajar o ingresar en otros entornos poco seguros o desconocidos.

Inicio verificado del sistema operativo Chrome


Chrome OS cuenta con un proceso grupal de reversión con integridad verificada, llamado inicio verificado. Al igual que con el arranque seguro, este proceso verifica la integridad del sistema operativo en uso. Si un atacante intenta manipular la máquina, el sistema lo detectará al reiniciarse y se negará a operar. El inicio verificado también impide que versiones vulnerables conocidas del sistema operativo se inicien, incluso si están firmadas.
El proceso de inicio verificado tiene cuatro componentes principales. Así es como funciona.
El primero de estos componentes es el firmware de solo lectura. La máquina Chrome OS se envía con este firmware que requiere modificación física de la máquina para admitir escritura a fin de que los atacantes remotos no puedan manipularlo. Este firmware de solo lectura contiene las claves criptográficas necesarias para verificar que el próximo componente haya sido firmado por una fuente de confianza y que tenga la cantidad mínima de código necesaria para ejecutar el siguiente componente en la cadena: el firmware de lectura y escritura. Este es un firmware que puede actualizarse automáticamente cuando sea necesario. Una vez que el firmware de solo lectura verifique que los contenidos provienen de una fuente confiable, se ejecutará el código de esta segunda pieza de firmware. Si, por el contrario, el firmware de solo lectura no puede verificar el contenido, el sistema intentará verificar la última copia de seguridad del firmware de lectura/escritura. Si la copia de seguridad de lectura/escritura se verifica, usará esa copia en su lugar. Si la copia de seguridad tampoco se verifica, invocará automáticamente el modo de recuperación, que es de solo lectura y no puede ser manipulado.
Por lo tanto, sabemos que si se ejecuta el firmware de lectura/escritura es porque el sistema verificó que proviene de una fuente confiable. El firmware de lectura/escritura se encarga de verificar el sistema operativo real de manera similar y verificará que el kernel provenga de una fuente confiable y no haya sido manipulado. Cuando se ejecute el kernel, verificará la integridad del sistema de archivos raíz que está usando. El sistema de archivos raíz contiene la instalación real del sistema operativo y los datos de usuario almacenados localmente. Entonces, si todo va bien, el firmware de solo lectura verifica y ejecuta el firmware de lectura/escritura. El firmware de lectura/escritura verifica y ejecuta el kernel, y el kernel monta y verifica el sistema de archivos raíz.
Chrome OS almacena el kernel y el sistema de archivos raíz en particiones separadas del disco duro, hay dos para cada una de ellas. La máquina actualmente usa particiones que son de solo lectura para que el sistema en ejecución no pueda modificarse. Las otras dos pueden ser modificadas por el software que realiza actualizaciones automáticas. Cuando se descarga una actualización, se implementa en las particiones que no están en uso actualmente. Cuando la máquina se reinicia, detectará la actualización, la verificará y se iniciará desde las particiones actualizadas. Cuando se reinicia, si la máquina detecta que uno de los kernels o de las las particiones del sistema de archivos están dañadas o fueron manipuladas, se iniciará desde el otro kernel o partición. Si ambos están en mal estado, pasará automáticamente al modo de recuperación.

Cifrado de datos del sistema operativo Chrome


Como señalamos anteriormente, Chrome OS guarda la mayoría de los datos de los usuarios en la nube, pero cierta información se almacena localmente para ayudar al desempeño y al uso sin conexión. Esta información incluye archivos descargados, marcadores, cookies y caché, entre otras cosas.
Chrome OS garantiza que solo el usuario pueda acceder a esta información cifrándola con la contraseña del usuario y el módulo de plataforma confiable, o TPM. Como recordatorio, el TPM es un dispositivo criptográfico que puede ser usado por el sistema operativo para garantizar la integridad del sistema y almacenar claves criptográficas para que los atacantes no puedan acceder a ellas. En Chrome OS, el TPM se usa solo para almacenar claves criptográficas.
En las nuevas generaciones de Chrome OS, el dispositivo criptográfico se llama H1. Tiene un propósito similar al del TPM y se podrá utilizar para habilitar aún más funciones de seguridad en el futuro.
Cuando un usuario accede por primera vez, Chrome OS usa una clave generada por el dispositivo criptográfico y la contraseña del usuario para crear un directorio encriptado para ese usuario específico, llamado Vault. Para desencriptar los datos del directorio, esa máquina específica debe iniciar Chrome OS correctamente y el usuario debe acceder con su contraseña. Si un atacante intentó extraer el disco y montarlo en una máquina diferente, no podrá acceder a ninguno de los datos, y como los datos están encriptados con la contraseña de cada usuario, las demás personas que usen la máquina tampoco podrán acceder a ella. Esto les da a los usuarios seguridad y privacidad. Si la función Powerwash está activada, la clave almacenada en el TPM se elimina junto con los datos borrados, lo que hace imposible que un atacante recupere la información de un disco duro.
Ahora bien, esto no significa que Chrome OS sea invulnerable. Todos los sistemas operativos tienen vulnerabilidades, por lo que siempre debemos tener en cuenta las prácticas de seguridad importantes. En el caso de Chrome OS, debemos estar alertas para evitar la instalación de extensiones maliciosas de Chrome que pueden robar nuestras contraseñas, por ejemplo, pero gracias a sus actualizaciones automáticas, la zona de pruebas, el inicio verificado, la encriptación de datos, Powerwash y el modo de recuperación fácil de usar, Chrome OS proporciona un modelo de seguridad en capas que lo convierte en el sistema operativo elegido por muchos expertos en seguridad.
