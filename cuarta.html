Aquí nos ocupamos de los formularios. Para definir un formulario lo etiquetamos con form. Esta etiqueta tiene dos atributos principales; action que indica donde enviamos los resultados, y method que indica como los enviaremos.


        <form action="/my-handling-form-page" method="post">
          <div>
            <label for="name">Name:</label>
            <input type="text" id="name" name="user_name">
          </div>
          <div>
            <label for="mail">E-mail:</label>
            <input type="email" id="mail" name="user_mail">
          </div>
          <div>
            <label for="msg">Message:</label>
            <textarea id="msg" name="user_message"></textarea>
          </div>
        </form>

Para la tarea de organizar y estructurar los elementos del formulario usaremos dos etiquetas; fieldset entre las que pondremos los elementos del formulario, y label que nos permite etiquetar estos elementos. Para añadir un título al grupo creado por fieldset utilizamos legend.

Un dato importante en label es su atributo for que nos permite asociarlo al atributo id del elemento del formulario, y así hacer seleccionable el texto de dicho elemento.


        <form>
          <fieldset>
            <legend>Fruit juice size</legend>
            <p>
              <input type="radio" name="size" id="size_1" value="small">
              <label for="size_1">Small</label>
            </p>
            <p>
              <input type="radio" name="size" id="size_2" value="medium">
              <label for="size_2">Medium</label>
            </p>
            <p>
              <input type="radio" name="size" id="size_3" value="large">
              <label for="size_3">Large</label>
            </p>
          </fieldset>
        </form>

Los elementos de entrada en un formulario en su mayoría son incluidos con la etiqueta input. Estos a su vez tiene una serie de atributos imprescindibles. Estos son;

  • autofocus que le da el foco al elemento que lo incluye.
  • disabled que deshabilita el elemento.
  • name que se manda con la información del formulario.
  • value que le da el valor inicial al elemento.

Entre los diferentes tipos de elementos de un formulario encontramos los de tipos texto. Todos con la etiqueta input pueden llegar a tener diferentes cometidos. Hay varios atributos que determinan el comnportamiento de las etiquetas;

  • readonly o disabled hace que sea de solo lectura o estar desactivado.
  • placeholder que da un valor de ejemplo.
  • size que le da el tamaño del elemento y lenght que da la cantidad de caracteres a introducir.

Para la entrada de texto usaremos input type="text"

<input type="text" id="comment" name="comment" value="I'm a text field">

Para una entrada de correo electrónico usamos input type="email". El atributo multiple nos permite enviar varias direcciones separadas por comas.

<input type="email" id="email" name="email" multiple>

Para un campo de contraseña utlizamos input type="password". Oculta los datos que tecleamos pero solo en la pantalla.

<input type="password" id="pwd" name="pwd">

Si queremos poner un campo de busqueda los hacemos con input type="search".

<input type="search" id="search" name="search">

Un campo de teléfono lo pondremos con input type="tel".

<input type="tel" id="tel" name="tel">

Un campo para introducir una dirección URL lo hacemos con input type="url".

<input type="url" id="url" name="url">

Los campos de texto con multiples lineas utilizan una etiqueta específica textarea que tiene su propia etiqueta de cierre. Para poner el texto por defecto en este caso se hace entre las dos etiquetas.

<textarea cols="30" rows="10">Texto por defecto</textarea>

Ahora veremos las listas desplegables para seleccionar uno o varios valores. La caja de selección de un solo valor se etiqueta con select y entre la etiqueta inicial y la de cierre la etiqueta option. En esta última podemos añadir el atributo selected para la opción por defecto. Además podemos agrupar varias opciones con la etiqueta optgroup, que visualiza el atributo label al abrir la lista. Si al elemento option le añadimos un atributo value, este contiene el valor que se envía en el formulario. Si no se envía el valor contenido en la etiqueta.


        <select id="groups" name="groups">
          <optgroup label="fruits">
            <option>Banana</option>
            <option selected>Cherry</option>
            <option>Lemon</option>
          </optgroup>
          <optgroup label="vegetables">
            <option>Carrot</option>
            <option>Eggplant</option>
            <option>Potato</option>
          </optgroup>
        </select>

Para una lista con la opción de seleccionar varias opciones tan solo añadiremos el atributo multiple. Estos no se muestran como un despegable, si no como una lista.


        <select multiple id="multi" name="multi">
          <option>Banana</option>
          <option>Cherry</option>
          <option>Lemon</option>
        </select>

Para crear una caja de texto que autocomplete el texto que escribimos, creamos una lista de valores con la etiqueta datalist con un listado creada entre etiquetas option. Esto lo asociamos a la etiqueta input con el atributo list.


        <label for="myFruit">What's your favorite fruit?</label>
        <input type="text" name="myFruit" id="myFruit" list="mySuggestion">
        <datalist id="mySuggestion">
          <option>Apple</option>
          <option>Banana</option>
          <option>Blackberry</option>
          <option>Blueberry</option>
          <option>Lemon</option>
          <option>Lychee</option>
          <option>Peach</option>
          <option>Pear</option>
        </datalist>

Una forma de hacer que esta forma de introducir texto sea retro-compatible sería añadir la etiqueta select dentro la la lista de valores, de forma que se muestre como desplegable en los navegadores que no soportan el autocompletado.


        <label for="myFruit">What is your favorite fruit? (With fallback)</label>
        <input type="text" id="myFruit" name="fruit" list="fruitList">
            
        <datalist id="fruitList">
          <label for="suggestion">or pick a fruit</label>
          <select id="suggestion" name="altFruit">
            <option>Apple</option>
            <option>Banana</option>
            <option>Blackberry</option>
            <option>Blueberry</option>
            <option>Lemon</option>
            <option>Lychee</option>
            <option>Peach</option>
            <option>Pear</option>
          </select>
        </datalist>
