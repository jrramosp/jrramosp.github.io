Añadiendo un icono a la página(favicon), un enlace a un estilo CSS y un enlace a un script


	<link rel="shortcut icon" href="favicon.ico" type="image/x-icon">
	<link rel="stylesheet" href="css/style.css">
	<script type="text/javascript" src="script.js"></script>

Para estructurar los documentos debeos utilizar los elemento HEADING: h1,h2,h3,h4,h5,h6

Usar listas, tanto numerales como listas abstractas 


	<ul>
		<li></li>
	</ul>
	<ol>
		<li></li>
	</ol>

Tambien definimos énfasis con <em></em> e importacia con <strong></strong>

  • <i> Se usa principalmente como itálica para: Palabras extranjeras, definiciones taxonomicas, terminos técnicos...
  • <b> Se usa principalmente como negrita para: palabras clave, nombre de productos...
  • <u> Se usa principalmente como subrayado para: nombre propio, error de escritura...

También podemos definir listas con definiciones, con la descripción primero y luego la definición


	<dl>
		<dt>Definición</dt>
		<dd>Descripción</dd>
	</dl>

Los enlaces deben contener descripciones lógicas (no un "Haz click aquí"), y son:

<a href="https://www.mozilla.org/en-US/">the Mozilla homepage</a>.

Se puede añadir el atributo download para enlaces de ficheros o recursos


	<a href="https://download.mozilla.org/?product=firefox-latest-ssl&os=win64&lang=en-US"
		download="firefox-latest-64bit-installer.exe">
	Download Latest Firefox for Windows (64-bit) (English, US)</a>

También podemos enlazar a un envío de correo


	<a href="mailto:nobody@mozilla.org?subject=Correo"></a>

Si queremos hacer enlaces a determinadas partes del documento primero se define la parte a enlazar con el atributo <id>


	<h2 id="Mailing_address">Mailing address</h2>

Depués el enlace en cuestión se hara a la descripción del anterior <id>


	<p>The <a href="#Mailing_address">company mailing address</a> can be found at the bottom of this page.</p>

O bien a un documento específico al que haremos referencia en el enlace


	<p>Want to write us a letter? Use our <a href="contacts.html#Mailing_address">mailing address</a>.</p>

Si citamos en nuestro texto en un elemento bloque lo haremos con


	<blockquote cite="https://developer.mozilla.org/en-US/docs/Web/HTML/Element/blockquote">
  	<p>The <strong>HTML <code>&lt;blockquote&gt;</code> Element</strong> (or <em>HTML Block Quotation Element</em>) 
  	indicates that the enclosed text is an extended quotation.</p>
	</blockquote>

Si citamos en un elemento lineal lo haremos


	<p>The quote element — <code>&lt;q&gt;</code> — is <q cite="https://developer.mozilla.org/en-US/docs/Web/HTML/Element/q">
	intended for short quotations that don't require paragraph breaks.</q></p>

Además tenemos el elemento cite para remarcar la cita


	<p>According to the <a href="https://developer.mozilla.org/en-US/docs/Web/HTML/Element/blockquote">
	<cite>MDN blockquote page</cite></a>:
	</p>

Para las abreviaturas utilizamos abbr


	<p>We use <abbr title="Hypertext Markup Language">HTML</abbr> to structure our web documents.</p>
	<p>I think <abbr title="Reverend">Rev.</abbr> Green did it in the kitchen with the chainsaw.</p>

Para los detalles sobre quien escribió la página web address es el elemento


	<address>
 	 <p>Page written by <a href="../authors/chris-mills/">Chris Mills</a>.</p>
	</address>

Para represntar Subscript y Superscript nada como sup ysub.

Si queremos mostar códigos de ordenador utlizaremos

  • <code>: For marking up generic pieces of computer code.
  • <pre>: For retaining whitespace (generally code blocks) — if you use indentation or excess whitespace inside your text, browsers will ignore it and you will not see it on your rendered page. If you wrap the text in <pre></pre> tags however, your whitespace will be rendered identically to how you see it in your text editor.
  • <var>: For specifically marking up variable names.
  • <kbd>: For marking up keyboard (and other types of) input entered into the computer.
  • <samp>: For marking up the output of a computer program.

Cuando añadimos una fecha conviene formatearla con la etiqueta <time>


	<time datetime="2016-01-20T19:30">20 January 2016, 7:30pm</time>

Secciones básicas en un documento HTML5

  • Cabecera: <header>.
  • Barra de navegación: <nav>.
  • Contenido principal: <main>, Con varias subsecciones representadas por <article>, <section>, y elementos <div>.
  • Barra lateral: <aside>; a menudo colocada en el <main>.
  • Pié: <footer>.

El documento iría estructurado de la siguiente forma:


	<html>
	<head>
	</head>
	<body>
		<header>
		</header>
		<nav>
		</nav>
		<main>
			<article></article>
			<aside></aside>
		</main>
		<footer>
		</footer>
	</body>	
	</html>

Para secciones específicas donde no está claro que poner, usaremos <div> y <span>

Crearemos saltos de página con la etiqueta <br> y líneas de división que denotan un cambio de temática con <hr>
